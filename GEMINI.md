# ClawBrain Project Constitution (GEMINI.md)

1. **AI-Only Closed Loop**: All code is generated by AI from `design/` specs. Bug fixes must update the spec first, then regenerate code. Manual intervention is prohibited.

2. **Self-Consistent Design Traceability**: Every spec must be a complete, self-contained functional specification — no dependency on prior code history. The first line of every source file must cite `# Generated from design/[file] v[ver]`.

3. **Phase-Based Audit Evidence**: Results are preserved independently per Phase. `results/` must contain associated log audit evidence with Input / Expected / Actual triples.

4. **Full Synchronous Updates**: Code, tests, data, and docs must change in lockstep. No partial updates.

5. **Canonical Naming Convention**: Filenames must be concise and include the phase identifier. Version numbers in filenames are prohibited.

6. **Centralised Test Data**: All static test data lives under `tests/data/`, named to match the Phase.

7. **Surgical Spec Correction**: When fixing a bug, modify only the specific logic in the design doc that caused the failure. Do not rewrite, rephrase, or touch sections that were already validated in prior tests. This preserves stable inheritance.

8. **English Everywhere**: Design documents, source code, inline comments, commit messages, and test output labels must all be written in English. This ensures the project is accessible to the widest possible audience and reviewable by any contributor.
