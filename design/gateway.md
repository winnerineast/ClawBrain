## 3. 强制性审计日志规范 (Side-by-Side Audit Standard)

### 3.1 对比布局 (Visual Comparison)
所有的测试日志必须严格遵循以下视觉布局进行输出：

```text
[AUDIT: Test Name]
------------------------------------------------------------
INPUT: <Input Data Snapshot>
------------------------------------------------------------
EXPECTED RESULT             | ACTUAL RESULT
--------------------------- | ----------------------------
<Expected Value>            | <Actual Value>
------------------------------------------------------------
MATCH: YES/NO
============================================================
```

### 3.2 针对长文本的处理 (Long Text Diff)
对于对话内容等长文本，EXPECTED 栏必须描述“核心验证点”或“期望的文本特征”，ACTUAL 栏必须打印对应的真实输出片段，以便人工肉眼对比。
